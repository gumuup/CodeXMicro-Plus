## 2.6.0 — 2026-07-20

### 面板交互修复

- Fast、Plan、推理旋钮和语音操作改为读取当前 `~/.codex/keybindings.json`，不再因 Codex 快捷键调整而失效。
- 摇杆支持直接点击四个方向，并将左右切换统一到 Codex 的上一/下一任务快捷键。
- 面板语音键改为点击开始、再次点击结束；物理按键映射继续保持按住/松开语义。
- 摇杆和语音在激活 Codex 后改走系统 HID 事件通道，修复 Electron 未接收定向 PID 事件但面板仍显示“成功”的假反馈。
- 非激活悬浮面板触发动作时强制将 Codex 切到前台，并等待系统确认后再发送 HID 事件，避免事件误发给微信等原前台应用。
- 摇杆左右改为调用 Codex 的 `Previous Chat / Next Chat` 原生菜单；语音键避开 macOS 通用“开始听写”菜单，改为读取 Codex 的 `composer.startDictation` 配置并发送带完整修饰键按下/释放序列的快捷键。
- 摇杆左右不再依赖 Codex 的标签页导航栈，改为在面板 A1–A6 的真实任务 ID 列表中循环并通过 `codex://threads/<id>` 打开上一个或下一个任务。

### 按键映射可靠性

- 单键与组合键统一改用 CGEventTap 一级事件监听，不再向系统申请 Carbon 独占热键；即使组合已被 macOS 或其他应用注册，也不会再显示“已被独占，当前未生效”。
- 组合映射完整处理按下、自动重复和松开事件，并继续过滤 CodeXMicro++ 自身合成的键盘事件，避免动作重复和回环触发。
- 新实例启动时会关闭相同 Bundle ID 的旧实例，防止多个监听器重复执行同一映射。
- 自定义按键设置页统一显示单键与组合键的辅助功能权限及监听状态。

### 构建与分发

- 应用版本、构建号、Codex 客户端标识、安装说明和通用 DMG 文件名统一更新为 2.6.0（构建 260）。

## 2.5.0 — 2026-07-19

### 物理按键映射

- 新增一级优先的物理按键映射路径，键盘任意单键均可直接绑定，包括 Home、字母、空格、Esc 与左右 Shift / Control / Option / Command。
- 命中的映射按键会拦截按下、自动重复和抬起事件；修饰键被映射后，其原生修饰标记也会从后续键盘事件中剥离，真正接管原按键功能。
- 单独轻点修饰键可录入修饰键本身，按住修饰键再按其他键则继续录入组合快捷键。
- 一级物理映射优先于使用同一按键或修饰键的组合快捷键；被覆盖的组合会明确标记为未生效。
- 带修饰键的组合继续使用 Carbon 全局热键，并改为独占注册，避免多个应用同时触发；若组合已被独占则拒绝新设置。
- 语音触发键统一为按住开始、松开结束，独立按键与组合快捷键均保留按下 / 抬起语义。
- 新增「自定义按键」设置页，集中展示监听权限、触发优先级、监听方式和全部动作绑定。

### 可靠性与隐私

- 组合键注册失败时原子回滚，避免新旧绑定同时丢失；重复判断改用键码与修饰键，不受键盘布局显示文本影响。
- 过滤自动重复和 CodeXMicro++ 合成事件，事件 tap 超时停用后会自动恢复。
- 更新辅助功能权限说明：键盘事件仅在内存中与用户绑定比较，未命中内容不保存或上传。

### 构建与分发

- 应用版本、构建号、Codex 客户端标识、安装说明和 DMG 文件名统一更新为 2.5.0（构建 250）。
- 本地运行与通用 DMG 构建在 `swift build` 因 Xcode License 不可用时，会回退到 Xcode 内置 Swift 6 工具链。
